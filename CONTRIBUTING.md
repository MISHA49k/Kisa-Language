## 🚀 Как начать

### 1. Fork репозиторий

Сделайте fork проекта на вашем GitHub аккаунте:
- Нажмите кнопку **Fork** в правом верхнем углу
- Клонируйте форк на вашу машину:

```bash
git clone https://github.com/ВАШ_USERNAME/Kisa-Language.git
cd Kisa-Language
```

### 2. Создайте ветку (branch)

```bash
git checkout -b feature/описание-фичи
```

**Правила для имен ветки:**
- `feature/` - для новых команд или функций
- `fix/` - для исправления ошибок
- `docs/` - для обновления документации
- `example/` - для добавления примеров

### 3. Внесите изменения

Отредактируйте файлы и добавьте свой вклад.

### 4. Протестируйте изменения

```bash
python kisa_interpreter.py
```

### 5. Commit и Push

```bash
git add .
git commit -m "Описание изменений"
git push origin feature/описание-фичи
```

### 6. Создайте Pull Request (PR)

- Перейдите на GitHub в свой форк
- Нажмите **Compare & pull request**
- Заполните описание PR
- Нажмите **Create pull request**

## 📝 Процесс контрибьюции

### Шаг 1: Обсуждение
- Откройте **Issue** перед началом большой работы
- Обсудите предложение в комментариях

### Шаг 2: Разработка
- Работайте в своей ветке
- Соблюдайте стандарты кода (см. ниже)
- Добавляйте комментарии к сложному коду

### Шаг 3: Тестирование
- Протестируйте на работоспособность
- Проверьте примеры в документации

### Шаг 4: Документирование
- Обновите README.md если нужно
- Добавьте примеры использования

### Шаг 5: Ревью
- Ждите ревью от мейнтейнеров
- Ответьте на замечания

### Шаг 6: Мерж
- После одобрения ваш код будет влит в main

## 🎯 Стандарты кода

### Python стиль

Следуйте **PEP 8**:

```python
# ✅ Правильно
def execute_command(command: VyveliCommand):
    """Выполняет команду выведи"""
    output = command.value
    print(output)


# ❌ Неправильно
def executeCommand(command):
    output=command.value
    print(output)
```

### Комментарии

```python
# ✅ Хороший комментарий
# Парсит координаты в формате (x,y)
def parse_coordinates(self) -> str:
    """Парсит координаты вида (100,200)"""
    self.expect(TokenType.СКОБКА_ОТКРЫВ)
    ...

# ❌ Плохой комментарий
# открыть скобку
self.expect(TokenType.СКОБКА_ОТКРЫВ)
```

### Именование переменных

```python
# ✅ Правильно
game_state = {}
object_name = "герой"
execute_igra()

# ❌ Неправильно
gs = {}
objName = "герой"
exc_igra()
```

## 📚 Типы контрибьюции

### 1. 🎮 Новые команды

Как добавить новую команду:

**Пример: команда "вычисли"**

#### Шаг 1: Добавить токен в lexer

```python
# kisa_lexer.py
class TokenType(Enum):
    ВЫЧИСЛИ = auto()
```

#### Шаг 2: Добавить ключевое слово

```python
# kisa_lexer.py - в методе __init__ класса Lexer
self.keywords = {
    ...
    'вычисли': TokenType.ВЫЧИСЛИ,
}
```

#### Шаг 3: Создать AST команду в parser

```python
# kisa_parser.py
@dataclass
class VychisliCommand(Command):
    """Команда: вычисли x плюс y."""
    operand1: str
    operation: str
    operand2: str
```

#### Шаг 4: Добавить парсер

```python
# kisa_parser.py
def parse_vychisli_command(self) -> VychisliCommand:
    """вычисли 5 плюс 3."""
    self.expect(TokenType.ВЫЧИСЛИ)
    operand1 = self.expect(TokenType.ЧИСЛО_ЛИТ).value
    operation = self.expect(TokenType.ПЛЮС).value
    operand2 = self.expect(TokenType.ЧИСЛО_ЛИТ).value
    self.expect(TokenType.ТОЧКА)
    return VychisliCommand(operand1, operation, operand2)
```

#### Шаг 5: Добавить исполняющий метод

```python
# kisa_interpreter.py
def execute_vychisli(self, command: VychisliCommand):
    """Выполняет команду вычисли"""
    val1 = int(command.operand1)
    val2 = int(command.operand2)
    
    if command.operation == "плюс":
        result = val1 + val2
    elif command.operation == "минус":
        result = val1 - val2
    
    msg = f"Результат: {result}"
    self.console_output.append(msg)
    print(msg)
```

#### Шаг 6: Добавить в execute метод

```python
# kisa_interpreter.py - в методе execute
elif isinstance(command, VychisliCommand):
    self.execute_vychisli(command)
```

## ❓ Часто задаваемые вопросы

### Q: С чего начать?
A: Посмотрите Issues с меткой `good-first-issue` или `help-wanted`.

### Q: Как добавить новую команду?
A: Следуйте 6 шагам в разделе "Новые команды".

### Q: Могу ли я изменить синтаксис?
A: Обсудите это в Issue перед изменениями.

### Q: Как долго ждать ревью?
A: Обычно 2-7 дней в зависимости от объема.

### Q: Что если мой PR отклонят?
A: Это нормально! Посмотрите комментарии и попробуйте еще раз.

### Q: Есть ли код стайл гайд?
A: Да, следуйте PEP 8 и смотрите примеры в коде.

### Q: Где я могу обсудить идеи?
A: На GitHub Issues или в нашем Discord сервере!
